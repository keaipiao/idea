## 1. 需求（对应 SOP 阶段 1）

### 1.1 做什么

一款轻量级想法捕捉工具，支持微信小程序和网页端，按「项目」维度组织想法，支持语音/文字输入，可勾选完成，多端同步。

### 1.2 为什么 / 用户价值

**目标用户**：个人开发者（自己），日常有大量碎片化想法需要记录。

**解决的问题**：
- 当前用"微信发给自己"记想法，但想法很快淹没在聊天记录里，无法找回
- 现有工具（Notion/语雀/flomo）要么太重（打开就要选页面），要么缺乏项目维度（纯流水记录）
- 走路、开会、洗澡时的灵感稍纵即逝，需要零摩擦的捕捉方式

**核心价值主张**：像发微信一样记想法，但有项目归属、有完成状态。

**不解决会怎样**：好想法继续丢失，项目进展散落各处无法追踪。

### 1.3 不做什么（scope 边界）

- 不做 AI 智能整理/分类（v2 迭代方向）
- 不做标签系统（MVP 数据量小，项目维度够用）
- 不做搜索功能（MVP 项目和想法数量有限，直接浏览）
- 不做团队协作/分享（纯个人工具）
- 不做提醒/通知推送
- 不做想法之间的关联/知识图谱
- 不做自动化工作流（GitHub Issue 同步等）
- 不做项目图标/封面（构思阶段项目可能连正式名称都没有）

### 1.4 成功指标

| 指标 | 目标 | 说明 |
|---|---|---|
| 自己每天使用 | ≥ 1 次/天 | 连续 2 周 |
| 想法不再丢失 | 0 次"想记但没记上" | 主观感受 |
| 输入耗时 | < 5 秒完成一条记录 | 从打开到发送 |
| 多端同步 | 小程序记、网页看，数据一致 | 无延迟感知 |

### 1.5 产品结构

```
创建项目 → 进入项目 → 记录想法（文字/语音）→ 勾选完成
```

**平台策略**：
- 微信小程序：主力输入端（随时随地记录）
- 网页端：主力整理端（电脑前回顾、批量操作）
- 两端共享同一后端 API，数据实时同步

## 2. 调研（对应 SOP 阶段 2）

### 2.1 微信登录 + unionid 机制

**小程序登录流程**：
1. 前端 `wx.login()` → 获取临时 `code`
2. 后端调用 `GET https://api.weixin.qq.com/sns/jscode2session`（参数：appid, secret, js_code, grant_type=authorization_code）
3. 返回：`openid` + `session_key` + `unionid`（需满足条件）

**unionid 获取条件**：小程序必须绑定到微信开放平台账号。绑定后 `code2Session` 直接返回 unionid，无需用户额外授权。

**网页扫码登录 OAuth 流程**（4 步）：
1. 构造授权链接引导用户扫码：`https://open.weixin.qq.com/connect/qrconnect?appid=APPID&redirect_uri=URI&response_type=code&scope=snsapi_login&state=STATE`
2. 用户扫码确认 → 回调地址带 `code`
3. 后端用 code 换 access_token：`GET https://api.weixin.qq.com/sns/oauth2/access_token`
4. 用 access_token 调 `/sns/userinfo` 获取用户信息（含 unionid）

**⚠️ 关键限制**：个人主体无法注册微信开放平台，因此个人主体小程序**无法获取 unionid**，也**无法使用网页扫码登录**。

**个体工商户认证**：
- 费用：300 元/次（年审续费）
- 周期：1-5 个工作日
- 材料：营业执照 + 经营者身份证 + 银行卡
- 认证后可绑定小程序 + 创建网站应用

### 2.2 微信同声传译插件

**接入方式**：
- 插件 AppID：`wx069ba97219f66d99`
- `app.json` 注册插件（version `0.3.6`），代码中 `requirePlugin('WechatSI')` 获取实例

**核心 API**：
```javascript
const manager = plugin.getRecordRecognitionManager()
manager.start({ lang: 'zh_CN' })  // 开始流式识别
manager.onRecognize = (res) => {}  // 中间结果回调
manager.onStop = (res) => {}       // 最终结果回调
manager.stop()                     // 停止识别
```

**免费额度**（无付费分层）：
| 接口 | 频率上限 | 每日上限 |
|---|---|---|
| 语音识别 ASR | 250 条/分钟 | 3 万条/天 |

**支持语言**：中文普通话、英语、粤语。安静环境识别率约 95%+。

**⚠️ 关键限制**：**个人主体小程序无法添加此插件**（插件类目要求企业主体）。

**替代方案**（若暂时无企业资质）：
- 腾讯云智能语音小程序插件（付费，无主体限制）
- 阿里云 ISI SDK（支持小程序端实时识别）
- `wx.getRecorderManager()` 录音 → 上传后端 → 调第三方 ASR API

### 2.3 Spring Boot + 微信 SDK 选型

**推荐 SDK**：WxJava（原 weixin-java-tools），GitHub 30k+ star，覆盖微信全场景。

**Maven 坐标**（v4.8.0，兼容 Spring Boot 3.5.3 + Java 21）：
```xml
<!-- 小程序模块 -->
<dependency>
    <groupId>com.github.binarywang</groupId>
    <artifactId>wx-java-miniapp-spring-boot-starter</artifactId>
    <version>4.8.0</version>
</dependency>
<!-- 开放平台模块（网页扫码登录） -->
<dependency>
    <groupId>com.github.binarywang</groupId>
    <artifactId>wx-java-open-spring-boot-starter</artifactId>
    <version>4.8.0</version>
</dependency>
```

**ORM 选型**：MyBatis-Plus（3 张表轻量 CRUD，Lambda 条件构造器、分页/逻辑删除内置、SQL 可控调试友好，国内 Java 生态主流）。

**项目基础结构**：
| 组件 | 选择 | 理由 |
|---|---|---|
| JDK | 21 LTS | 虚拟线程、Pattern Matching |
| Framework | Spring Boot 3.5.3 | 长期支持 |
| 连接池 | Druid | 阿里开源，监控面板 + SQL 防火墙 |
| ORM | MyBatis-Plus | 轻量 CRUD 最优解 |
| 构建 | Maven | 稳定 |

### 2.4 竞品分析

| 产品 | 定位 | 优势 | 不足（我们的差异点）|
|---|---|---|---|
| **flomo** | 卡片式灵感记录 | 极简、微信输入、标签 | 无项目维度、无完成状态、纯流水账 |
| **滴答清单** | 全功能待办管理 | 功能全面、多端 | 太重、记个想法要选列表/标签/优先级 |
| **闪念贝壳** | 语音优先 + AI | 语音识别好、AI 整理 | 偏 AI 概念、无项目组织 |
| **Memos** | 开源自部署笔记 | 开源免费、可自部署 | 需要运维、无小程序、无语音 |
| **Apple 备忘录** | 系统自带 | 零安装 | 无项目维度、iOS 生态内 |

**我们的差异化**：项目维度 + 完成状态 + 微信生态（小程序随手记）+ 零学习成本。本质上是 flomo 的"轻"+ 滴答的"完成追踪"的交叉点。

### 2.5 关键不确定性 & 决策更新

**⚠️ 硬前置条件确认**：个体工商户是本项目的阻断性依赖——没有它，三大核心能力全部不可用：

| 能力 | 依赖 | 备选降级方案 |
|---|---|---|
| 多端账号关联（unionid）| 开放平台认证 | 开发期用手机号关联（需短信服务） |
| 网页扫码登录 | 开放平台网站应用 | 开发期用账号密码登录 |
| 语音输入（同声传译）| 企业主体小程序 | 录音 + 后端调腾讯云 ASR |

**ADR 更新**：
- **决策 5**（新增）：开发期采用"双轨策略"——先用降级方案（文字输入 + 测试 token）跑通全流程，个体工商户认证通过后切换正式方案。不因资质问题阻塞开发进度。
- **决策 6**（新增）：语音方案最终确认用同声传译插件（免费、30k 条/天够用）。若认证后仍遇插件审核问题，备选方案为 `wx.getRecorderManager()` + 腾讯云 ASR API（一句话识别免费额度：每月 5000 次）。

### 4.4 关键决策（ADR）

- **决策 1**：网页端用微信扫码登录，不用账号密码。原因：统一身份体系，用 unionid 关联，用户无需记密码。前置条件：需个体工商户认证微信开放平台。
- **决策 2**：语音识别用微信同声传译插件（AppID: `wx069ba97219f66d99`），不接第三方 ASR。原因：免费（3 万条/天）、流式识别体验好、接入简单。限制：仅小程序端可用，网页端 MVP 不做语音。备选：`wx.getRecorderManager()` + 腾讯云 ASR。
- **决策 3**：完成状态用 `completed_at` 字段而非 `is_completed` 布尔值。原因：零冗余，NULL 判断即可，且自动记录完成时间。
- **决策 4**：不做 source_type 字段区分语音/文字来源。原因：最终存储都是文字，来源是交互层概念，不需要持久化。
- **决策 5**：开发期采用"双轨策略"。先用降级方案（纯文字输入 + 测试 token 认证）跑通全流程，不因个体工商户认证周期阻塞开发。认证通过后切换微信正式登录 + 语音插件。
- **决策 6**：ORM 选 MyBatis-Plus 而非 JPA。原因：3 张表轻量 CRUD 场景下 MyBatis-Plus 更轻、SQL 可控、调试友好，且符合项目全局规范（Java 21 + Spring Boot + Maven + MyBatis-Plus）。
- **决策 7**：微信 SDK 选 WxJava 4.8.0。原因：国内最成熟的微信 Java SDK（30k+ star），覆盖小程序 + 开放平台全场景，提供 Spring Boot 3.5.3 Starter 开箱即用。

### 4.6 风险与回滚边界

| 风险 | 影响 | 概率 | 应对 |
|---|---|---|---|
| 个体工商户办理周期 | 阻塞 unionid + 扫码登录 + 语音三大能力 | 中 | 双轨策略：开发期用降级方案，认证后切换 |
| 同声传译插件审核不通过 | 语音功能不可用 | 低 | 备选：录音上传 + 腾讯云 ASR API |
| WxJava 4.8.0 与 Spring Boot 3.5.3 Jakarta 兼容问题 | 启动报错 | 低 | 手动配置 Bean 绕过 Starter，或降级到 4.6.x |
| 小程序审核不通过 | 无法发布 | 低 | 选"工具 > 效率"类目，功能纯工具类合规 |
| 微信接口调用频率限制 | 登录/语音高峰期受限 | 极低 | 个人使用量级远低于限额 |

**回滚边界**：后端 + 数据库部署在自有服务器，出问题可 5 分钟回滚到上一个 docker 版本。小程序发布后回滚需提交新版本审核（约 1-2 小时）。
