## 1. 需求(PR-1 user story)

### 1.1 做什么

为整个 IdeaBox 项目搭建可运行的后端基础:
- Spring Boot 3.5.3 + Java 21 + Maven 项目骨架
- PostgreSQL 16 + Flyway migration + 3 张表(`t_user` / `t_project` / `t_idea`)
- 9 个 REST endpoint(项目/想法 CRUD + 开发期登录 + 当前用户)
- **开发期 JWT 认证机制**:绕过微信登录,通过 `POST /api/auth/login/dev` 直接拿测试用户 JWT。后续 PR-4 接入真微信登录时复用同一套 JwtUtil + Filter,只换签发入口

### 1.2 为什么 / 用户价值

- **解锁前端开发**:PR-2(小程序)/ PR-3(网页)阻塞在后端 API 之前;PR-1 完成后两者可立即并行启动
- **解耦微信资质阻塞**:外层 ADR-5 的"双轨策略"在 PR-1 落地。开发期 JWT 让 PR-2/PR-3 不依赖个体工商户认证
- **建立项目骨架契约**:包结构、Result wrapper、异常处理、配置约定一次性定型,后续 PR 沿用

### 1.3 不做什么(PR-1 scope 边界)

- ❌ 微信登录(留 PR-4) — 仅留接口位置 `/api/auth/login/mp` 占位 501
- ❌ 前端任何代码(PR-2 / PR-3)
- ❌ 生产环境部署 — 仅本地 `docker compose` 起 Postgres + `mvn spring-boot:run` 跑 backend
- ❌ Druid 监控面板对外暴露 — dev profile 开,prod profile 关
- ❌ 限流 / 风控 / 审计日志

### 1.4 PR-1 验收标准

| 项 | 标准 |
|---|---|
| 启动 | `mvn spring-boot:run` 30s 内 `Started IdeaboxApplication`,无 `ERROR` |

| 健康 | `curl /actuator/health` → `{"status":"UP"}` |

| 业务 endpoint | 9 个 endpoint 全部通过 curl 验证(创建项目 → 查项目 → 创建想法 → 完成想法) |
| 单测 | Service 层全绿 |
| 集成测试 | `@SpringBootTest contextLoads()` + JWT 端到端 + Service↔Mapper↔Postgres(Testcontainers) |
| 质量门 | `/review` PASS + `/cso` 0 P0/P1 + 真实启动验证全 ✅ |

### 4.2 数据模型

**全局约定**:
- 主键 `BIGSERIAL`(走 PG sequence)
- 时间戳全用 `TIMESTAMP`(无时区,后端统一按 Asia/Shanghai 处理)
- 表 / 字段 **必须中文 COMMENT**(全局规则 11)
- 软删:**MVP 不做**,所有表硬删

#### Flyway `V1__create_tables.sql`

```sql
-- ========== 用户表 ==========
CREATE TABLE t_user (
  id BIGSERIAL PRIMARY KEY,
  union_id VARCHAR(64),
  open_id_mp VARCHAR(64),
  open_id_web VARCHAR(64),
  nickname VARCHAR(64),
  avatar_url VARCHAR(512),
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
COMMENT ON TABLE t_user IS '用户表 - 多端身份关联';
COMMENT ON COLUMN t_user.union_id IS '微信开放平台 unionid(PR-4 接入后填充)';
COMMENT ON COLUMN t_user.open_id_mp IS '小程序 openid';
COMMENT ON COLUMN t_user.open_id_web IS '网页应用 openid';
COMMENT ON COLUMN t_user.nickname IS '微信昵称';
COMMENT ON COLUMN t_user.avatar_url IS '头像 URL';
CREATE UNIQUE INDEX uk_user_union_id ON t_user(union_id) WHERE union_id IS NOT NULL;
CREATE UNIQUE INDEX uk_user_open_id_mp ON t_user(open_id_mp) WHERE open_id_mp IS NOT NULL;
CREATE UNIQUE INDEX uk_user_open_id_web ON t_user(open_id_web) WHERE open_id_web IS NOT NULL;

-- ========== 项目表 ==========
CREATE TABLE t_project (
  id BIGSERIAL PRIMARY KEY,
  user_id BIGINT NOT NULL REFERENCES t_user(id) ON DELETE CASCADE,
  name VARCHAR(100) NOT NULL,
  sort_order INT NOT NULL DEFAULT 0,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
COMMENT ON TABLE t_project IS '项目表';
COMMENT ON COLUMN t_project.user_id IS '所属用户 id';
COMMENT ON COLUMN t_project.name IS '项目名称';
COMMENT ON COLUMN t_project.sort_order IS '排序序号(数值小靠前)';
CREATE INDEX idx_project_user ON t_project(user_id, sort_order);

-- ========== 想法表 ==========
CREATE TABLE t_idea (
  id BIGSERIAL PRIMARY KEY,
  project_id BIGINT NOT NULL REFERENCES t_project(id) ON DELETE CASCADE,
  content TEXT NOT NULL,
  completed_at TIMESTAMP,
  sort_order INT NOT NULL DEFAULT 0,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
COMMENT ON TABLE t_idea IS '想法表';
COMMENT ON COLUMN t_idea.project_id IS '所属项目 id';
COMMENT ON COLUMN t_idea.content IS '想法内容(纯文本)';
COMMENT ON COLUMN t_idea.completed_at IS '完成时间(NULL=未完成,有值=已完成)';
COMMENT ON COLUMN t_idea.sort_order IS '排序序号';
CREATE INDEX idx_idea_project ON t_idea(project_id, completed_at, sort_order);
```

### 4.3 接口契约

**统一约定**:
- 所有响应包 `Result<T> = { code: int, message: string, data: T }`,业务成功 `code=0`
- 所有受保护 endpoint 必须带 `Authorization: Bearer <jwt>`,缺失返 401
- `userId` 从 JWT 解出后存入 `UserContext.ThreadLocal`,Service 层直接读
- 时间字段输出 ISO-8601(`2026-05-28T10:30:00`)

| 方法 | 路径 | 鉴权 | 入参 | 出参 |
|---|---|---|---|---|
| POST | `/api/auth/login/dev` | ❌ | `{userId?: 1}` | `{token, expiresAt, user}` |
| POST | `/api/auth/login/mp` | ❌ | — | **501 Not Implemented**(PR-4 实装) |
| GET | `/api/users/me` | ✅ | — | `User` |
| GET | `/api/projects` | ✅ | `?page=1&size=20`(默认全量,可分页) | `PageResult<ProjectVO>` |
| POST | `/api/projects` | ✅ | `{name}`(`@NotBlank @Size(max=100)`) | `ProjectVO` |
| PUT | `/api/projects/{id}` | ✅ | `{name?, sortOrder?}` | `ProjectVO` |
| PUT | `/api/projects/reorder` | ✅ | `{ids: [Long]}`(按数组顺序设 sortOrder) | `null` |
| DELETE | `/api/projects/{id}` | ✅ | — | `null`(级联删想法) |
| GET | `/api/projects/{id}/ideas` | ✅ | `?page=1&size=50&completed=false`(按状态过滤可选) | `PageResult<IdeaVO>` |
| POST | `/api/projects/{id}/ideas` | ✅ | `{content}`(`@NotBlank @Size(max=5000)`) | `IdeaVO` |
| PUT | `/api/ideas/{id}` | ✅ | `{content?, completed?, sortOrder?}` | `IdeaVO` |
| PUT | `/api/projects/{projectId}/ideas/reorder` | ✅ | `{ids: [Long]}` | `null` |
| DELETE | `/api/ideas/{id}` | ✅ | — | `null` |

**鉴权与所有权校验**:
- 所有 `/api/projects/**` `/api/ideas/**` 操作必须校验资源 owner === `UserContext.userId`,否则返 403(`code=403001`)
- DELETE 用硬删 + 外键级联(`ON DELETE CASCADE`)
- DELETE 不存在的 id 返 404(`code=404001`,而非静默成功)

**分页响应包**:

```json
{ "code": 0, "message": "ok", "data": {
  "records": [...], "total": 42, "page": 1, "size": 20, "pages": 3
}}
```

**HTTP 状态码 ↔ Result.code 映射**:

| HTTP | 业务 code | 场景 |
|---|---|---|
| 200 | 0 | 成功 |
| 400 | 400001 | 参数校验失败(`@Valid` 触发) |
| 401 | 401001 | JWT 已过期 |
| 401 | 401002 | JWT 签名无效或解析失败 |
| 401 | 401003 | Authorization 头缺失 |
| 403 | 403001 | 非资源所有者 |
| 404 | 404001 | 资源不存在 |
| 500 | 500000 | 系统内部异常(兜底) |

所有异常**统一经 `GlobalExceptionHandler` 转 `Result`,前端只解析 `code` 字段**,HTTP 状态码仅作传输层提示。

### 4.4 关键决策(PR-1 ADR)

- **PR-1 ADR-1**:开发期 JWT 用 HMAC-SHA256(对称) — 单机部署不需要 RSA。secret 从 `IDEABOX_JWT_SECRET` 环境变量读,长度 ≥ 32 字节,缺失时启动 fail-fast
- **PR-1 ADR-2**:`UserContext` 用 `ThreadLocal<Long>` 存当前 userId,JwtAuthFilter 入口 set,filter chain 末尾 remove。**禁止跨线程任务携带**(目前无异步,后续 PR 引入 `@Async` 时需要 `TransmittableThreadLocal` 升级)
- **PR-1 ADR-3**:接口契约采用 `completed: boolean`(请求侧)而非 `completedAt: timestamp` — 前端心智简单。Service 层翻译为 `completed_at = completed ? now() : null`
- **PR-1 ADR-4**:Result wrapper 强制 `data` 字段存在(null 也行),前端解构稳定
- **PR-1 ADR-5**:dev profile 启用 Druid Web 监控(`/druid/**` localhost only);prod profile 关闭 — 避免暴露 SQL 给外网
- **PR-1 ADR-6**:JWT 有效期 7 天(168h),不实现 refresh token(MVP 个人用,过期重登成本可忽略)
- **PR-1 ADR-7**:Flyway 版本号采用 `V<整数>__<下划线分词>.sql` 风格(`V1__create_tables.sql` / `V2__seed_dev_user.sql`),便于排序
- **PR-1 ADR-8**:`/api/auth/login/dev` 仅在 `spring.profiles.active` 包含 `dev` 时注册 — prod profile 启动时该 endpoint 不存在(`@Profile("dev")`),防止生产被绕过登录
- **PR-1 ADR-9**:HTTP 状态码 ↔ Result.code 双层契约。HTTP 仅传输层提示,前端只解析 `Result.code`。详见 § 4.3 映射表
- **PR-1 ADR-10**:CORS。dev profile `allowedOrigins: ["http://localhost:3000", "http://localhost:5173"]` + `allowedMethods: *` + `allowedHeaders: *` + `allowCredentials: true`;prod profile 留 PR-3 收紧(白名单网页域名)。**小程序请求 wxapp 协议无 Origin 头,不受 CORS 约束**
- **PR-1 ADR-11**:输入校验用 Jakarta Bean Validation。Request DTO 字段强制 `@NotBlank` + `@Size`:`name` ≤100 字,`content` ≤5000 字(覆盖典型语音转写一段)。`GlobalExceptionHandler` 捕 `MethodArgumentNotValidException` → `Result.code=400001` + 字段级错误列表
- **PR-1 ADR-12**:事务边界。`@Transactional` 默认放 **Service 方法级**(不放类级,避免只读方法被裹事务);只读查询显式 `@Transactional(readOnly = true)` 减锁开销;Controller 不带事务
- **PR-1 ADR-13**:DTO 转换约定。**Controller 层手写 `XxxVO.fromEntity(e)` + `XxxCreateReq.toEntity()` 静态方法**,不引 MapStruct(避免编译期注解处理器复杂度);Service 接 entity 返 entity,Service 内部不知道 VO/DTO 存在(便于复用)
- **PR-1 ADR-14**:Lombok 引入。`@Data`(entity 默认) + `@Builder`(DTO) + `@RequiredArgsConstructor`(Service / Controller 注入)。`pom.xml` 加 `<scope>provided</scope>`;IDE 必装 Lombok 插件
- **PR-1 ADR-15**:`created_at` / `updated_at` 用 **MyBatis-Plus `MetaObjectHandler`** 自动填充(`@TableField(fill = INSERT)` / `INSERT_UPDATE`),不用 PG trigger。理由:① 全 Java 路径可单测 ② Flyway 迁移文件干净不含触发器
- **PR-1 ADR-16**:分页用 **MyBatis-Plus `PaginationInnerInterceptor`**;Controller 返 `PageResult<T>`(records / total / page / size / pages)。即使 MVP 数据量小,接成本 < 1h 且为 PR-3 网页端"加载更多"预留

### 4.7 风险与回滚边界

| 风险 | 影响 | 应对 |
|---|---|---|
| Spring Boot 3.5.3 BOM 版本与依赖不兼容 | 启动 fail | 阶段 3 第一步 `mvn dependency:tree` 看冲突,必要时 BOM exclude |
| jjwt 0.12 API 与旧 0.11 教程不一致 | 写错签发逻辑 | 阶段 3 严格按 jjwt 0.12 官方 quickstart 写,不抄旧博客 |
| Druid Spring Boot 3 starter 与 Jakarta 命名空间冲突 | 启动 fail | 必须用 `druid-spring-boot-3-starter`(带 `-3-`) |
| MyBatis-Plus + PG `BIGSERIAL` 主键回填 | INSERT 后 entity.id 仍为 null | `@TableId(type = IdType.AUTO)` 显式指定 |
| 本地无 docker 环境 | 起不来 Postgres | 用户已有 Docker Desktop;阶段 5 验证 |
| `MetaObjectHandler` 注册不生效 | `created_at` / `updated_at` 入库 NULL | `@Component` + `MybatisPlusConfig` 注入,集成测试断言时间戳非空 |
| `JwtAuthFilter` 异常没被捕获 | 401 返默认 HTML 而非 Result JSON | filter 内 try-catch 写 `response.getWriter().write(Result.fail(401001).toJson())`,集成测试 NEG 6 case 兜底 |
| 重排 batch update 死锁 | 高频拖拽时两个事务互锁 | Service 内按 id 升序排序后批量 update,死锁概率降到 0;事务隔离用默认 `READ_COMMITTED` |

**回滚边界**:PR-1 落地前项目无任何代码 / git 历史。出问题直接删 `pom.xml` + `src/` 重来,或 `git reset --hard <init commit>`。
