## 1. 需求(PR-3 specific,阶段 0)

### 1.1 PR-3 做什么(一句话)

用 **Next.js 16 + TypeScript + Tailwind 4** 实现网页端的项目管理 + 想法记录,**复用 PR-1 后端 13 个 endpoint**,登录走开发期 JWT(`/api/auth/login/dev`),桌面端双栏 + 移动端单栏响应式。

### 1.2 与 feature 级 user story 的关系

| 父级总览 | PR-3 承担 | PR-3 不承担 |
|---|---|---|
| 文字输入 ✅ | 文字输入框 + Enter 发送 | 语音(PR-5 单独做小程序端,网页 MVP 不做语音) |
| 多端同步 ✅ | 网页端 CRUD,与小程序共享同一后端 | 实时推送(无 WebSocket,刷新 / 重连方案后续看痛点) |
| 微信登录 ⏳ | **暂用 dev login**(MVP),前端预留 `useAuth` hook 抽象未来扩展 | 微信扫码(PR-4 做,需个体工商户认证后) |
| 项目维度组织 ✅ | 左侧栏项目列表 + 右侧主区域想法列表 | 项目归档 / 标签(远期) |
| 完成状态 ✅ | 点击 ○ 完成 + 已完成区折叠 | 完成统计 / 时间趋势(远期) |

### 1.3 PR-3 不做什么(scope 边界)

- ❌ **不做微信扫码登录**:需开放平台资质,留 PR-4。MVP 用 dev login 跑通体验。
- ❌ **不做语音输入**:网页端语音留远期(`navigator.mediaDevices` + 后端 ASR,需付费 API)。
- ✅ **做 IdeaItem 拖拽排序**(用户 2026-05-28 评审追加):引入 `@dnd-kit`,IdeaItem 左侧拖拽手柄,松手触发 PR-1 `PUT /api/projects/{id}/ideas/reorder`。ProjectSidebar 不做拖拽(项目数量少 + 改顺序低频)。
- ❌ **不做实时推送 / WebSocket**:用户切回页面或操作时拉取最新数据即可,MVP 数据量小。
- ❌ **不做暗色模式**:UI 规范默认浅色,暗色留远期(`next-themes` 注入成本低,需要时再加)。
- ❌ **不做 SSR**:整个 app 用 Client Components + Next.js App Router,首屏交互依赖 JWT,SSR 无收益反而复杂化。`app/page.tsx` 用 `'use client'`,生产部署用 `next start` 但实际只起 client-side rendering。
- ❌ **不做 Tailwind 之外的 UI 库**:不引入 shadcn/ui / antd / mui。MVP 5 个页面手写组件足够,引入组件库后期反而难定制。

### 1.4 成功指标

| 指标 | 目标 | 验证手段 |
|---|---|---|
| 桌面端首屏到记录耗时 | < 5 秒 | 联调实测(打开 → 输入 → 发送) |
| 移动端响应式 | iPhone 12(390×844)+ iPad(810×1080)布局不破 | DevTools device mode |
| 与小程序数据一致 | 网页改完 → 小程序刷新看到 / 反之 | 跨端联调(PR-2 启动后回归) |
| 0 console error | Chrome DevTools console 0 红 | 联调阶段 5 验证 |
| Lighthouse 性能 ≥ 80 | desktop & mobile | 远期优化项,MVP 不卡死 |

### 1.5 关联变更

- **后端**:不动 PR-1 代码,纯消费 13 个 endpoint。若联调发现 gap → 回流 PR-1 改但走 hotfix 子 PR。
- **小程序**:无依赖。PR-2 启动时复用 PR-3 跑通的 API 契约。
- **部署**:新增 nginx 反代规则(根 `/` → `web/`,`/api/*` → backend)、新增 `web/Dockerfile` + `docker-compose.prod.yml` 节点、新增环境变量 `NEXT_PUBLIC_API_BASE_URL` + `NEXT_PUBLIC_AUTH_MODE`。
- **数据库**:不改 schema。**PR-1 PostgreSQL dev 端口 5432 → 25432**(避本机其他 PG 实例冲突);若未来引入 Redis 用 26380。同步改 `docker-compose.dev.yml` + `application-dev.yml` + `.env.example`(本 PR-3 顺手做)。

### 1.6 上线策略(Outside voice E 应答)

**PR-3 本身不部署生产**:dev login 是开发期降级方案,prod 启动时 `/api/auth/login/dev` endpoint 不存在(PR-1 `@Profile("dev")`)。PR-3 落地的真实意义:
- ① 验证 Next 16 + Tailwind 4 + @dnd-kit + SWR 在真服务器环境跑通(本机 prod stack 预演,Outside voice I)
- ② 把 API 客户端 `web/lib/api.ts` + types 跑通 → PR-2 小程序直接抄思路
- ③ 提供 dev 期内部使用工具 → 作者自己用先,反馈给 PR-4 微信扫码做接口收敛
- ④ 锁定 Dockerfile + nginx + docker-compose.prod.yml 框架,**PR-4 微信扫码完成时一并 push 到生产** + 切 `NEXT_PUBLIC_AUTH_MODE=wechat`

**为什么不先 PR-2**(阶段 0 用户决定):Chrome 真浏览器联调质量门 BLOCKING 可控,小程序需要用户陪做开发者工具点击。先 PR-3 跑通契约再 PR-2 复用。

## 2. 调研(对应 SOP 阶段 1,已联网核实 2026-05-28)

### 2.1 ctx7 验证的版本与最佳实践

**Next.js**(`/vercel/next.js`,2026-05-28 最新 `v16.2.2`,GA 状态):
- App Router 已是默认推荐(Pages Router 仅 legacy)
- `output: 'standalone'`(`next.config.ts`)用于 Docker:Build 阶段生成自包含 `.next/standalone/server.js`,运行 `node server.js` 代替 `next start`,镜像可 ≤ 200MB
- 官方 Dockerfile 三阶段模板(`deps → builder → runner`)使用 `NODE_VERSION=24.13.0-slim`,我们用 `node:22-alpine`(本机 Node v22.22.2,LTS 兼容)
- Client Component 中可直接 `useEffect` 读 `localStorage`,SSR 阶段 useEffect 不执行 → 无 hydration mismatch
- 内置 `useReportWebVitals` 可监 Web Vitals(MVP 跳过)

**Tailwind CSS**(`/tailwindlabs/tailwindcss.com`,2026-05 最新 `v4.3`):
- **v4 安装大幅简化**:`npm install tailwindcss @tailwindcss/postcss`(postcss-import + autoprefixer 不再需要)
- `postcss.config.js`:`{ plugins: ["@tailwindcss/postcss"] }`
- CSS 入口:`@import "tailwindcss";`(替代 v3 的 `@tailwind base/components/utilities` 三行)
- **`@theme {}` 自动生成 CSS variables**(`--color-primary` 等),零配置和 design token 系统集成
- **不再需要 `tailwind.config.js`**(v4 可全用 CSS 配,但保留 JS 配置仍兼容)

**关键 API 调整**:
- 我们用 fetch + 手写 hook(无 SWR / TanStack Query),理由:MVP 3 个数据模型,getter 函数 + useState + useEffect 模式手写 10 行就够
- 用 `'use client'` 全 client component(ADR-1 已锁)
- `useEffect` 内 `localStorage.getItem('ideabox.jwt')` 拿 token,首次 render 用 null 状态(渲染 skeleton),token 拿到后触发 fetch

### 2.2 firecrawl / WebSearch 行业做法对照

**业界共识**(2026-05 时间点):
- **flomo / 滴答 / Memos 网页端均用 client-rendered SPA**,首屏 fetch + skeleton,非 SSR(JWT-aware,SSR 收益小)
- **localStorage JWT 是 MVP 阶段主流**,httpOnly cookie 等到引入微信扫码(标准 OAuth flow)时再切
- **Tailwind 4 + Next.js 16** 组合在生产项目已稳定(StackOverflow 反馈无 Windows 启动问题,Node ≥ 20)

**竞品布局对照**:
- flomo:**单栏**(无项目维度,流水账模式)— 我们的差异化体现
- 滴答清单:**三栏**(列表 / 任务 / 任务详情)— 我们简化到双栏,因为想法本身没有"详情页"概念
- Memos:**双栏**(自托管开源)— 与我们结构最接近

### 2.3 项目内既有相似模式

**首次前端,无既有可复用**。新建标准结构:

- `web/lib/api.ts`:fetch 包装,带 Authorization header + Result.code 错误解析
- `web/lib/auth.ts`:dev login + token 持久化(`localStorage`)
- `web/app/_components/`:私有组件
- `web/app/_hooks/`:`useAuth` / `useProjects` / `useIdeas`

**后端契约对齐(Explore agent 深扫报告,2026-05-28)**:

| 类别 | 数量 | 关键约束 |
|---|---|---|
| Endpoints | **13 个**(我之前算 11 漏了 `/api/users/me` + `/api/auth/login/mp` 占位) | 12 个需 JWT,1 个无需(`/api/auth/login/dev`) |
| DTO 校验规则 | 6 个 RequestBody | `@NotBlank` / `@Size(max=N)` / `@Pattern(非空白)` / `@Positive` / `@NotEmpty` |
| ResultCode | 9 档 | `0/400001/401001/401002/401003/403001/404001/501000/500000` |
| CORS 允许 Origin(dev) | 4 个 | `localhost:3000`, `localhost:5173`, `127.0.0.1:3000`, `127.0.0.1:5173` |
| Authorization Header | `Bearer <token>` | dev 用户 id=1 种子,过期 7 天 |

完整 endpoint / DTO / VO / ResultCode / CORS 详情见 PR-1 源码 + 阶段 1 Explore 报告(直接落到 `web/lib/types.ts`)。

### 3.5.2 关键交互决策

- **删除不弹 modal**:点删除立即触发 `deleting` 状态 + toast undo(5s)。Krug "Don't make me think" — 二次确认是负担。后端 DELETE 在 toast 过期才真发(若用户点 undo 则取消 fetch);PR-3 ADR-10。
- **IdeaInput 乐观更新**:**破 ADR-2 一个例外** — 单条 idea 插入立刻乐观渲染(占位 + spinner),失败时变红 + 重试。理由:输入是高频操作,200ms 等待会累。ADR-2 仍适用于编辑 / 重排 / 批量删除等低频写。新增 PR-3 ADR-11 沉淀此例外。

### 3.5.3 已完成区折叠规则

- 已完成想法折叠在底部分隔线下方,默认收起
- 分隔线文案:"── 已完成 (N) ──",点击展开 / 收起
- 展开时显示已完成卡片,文字色 `--text-secondary`,完成时间显示在右下角灰字

### 3.5.4 时间维度情感弧(Pass 3 修法,最小反馈策略)

| 时间窗口 | 用户状态 | 反馈策略 |
|---|---|---|
| 首次 5 秒 | 0 项目 0 想法 | EmptyState 居中"还没有项目,点击 + 开始" |
| 首次 5 分钟 | 创建 + 输入第一条 | 仅"已记录" toast(2s),无庆祝、无引导 |
| 1 周 / 50 想法 | 数据增长 | 完成区折叠默认收起;统计可见(项目卡显示数量) |
| 1 月 / 100+ | 数据膨胀 | (后续 PR 评估搜索 / 标签必要性,MVP 阶段不预先优化) |

**核心原则**:这是个人工具不是 to-C 产品,**克制 engagement**。Norman 三层只走「行为层」(用就完成任务),不做「反思层」(里程碑 / 成就感)。

**明确不做**(避免 engagement 设计漂移):
- ❌ 首次创建项目 toast 庆祝
- ❌ 每日"今天 N 个未完成"提醒
- ❌ 第 N 条想法里程碑动画
- ❌ 完成区文案 emoji

### 3.7 设计系统对齐(Pass 5 修法)

**单一权威源**:`docs/_UI设计规范.md` 是项目级唯一 design token 来源,涵盖 8 色 / 3 档圆角 / 5 档间距 / 字体栈 / 通用状态。

**PR-3 实施约束**:
- ✅ 仅消费 `_UI设计规范.md` 现有 token,**不新增**任何 token(颜色 / 圆角 / 间距 / 字体)
- ✅ Tailwind 4 `@theme` 块在 `globals.css` 镜像 `_UI设计规范.md` 全部 token,**作为 CSS variable 源**;不重复定义颜色值

### 4.3 数据流(单 user state)

无全局状态管理库(Redux / Zustand),用 Next.js + React 19 内置:

| 数据 | 存储 | 生命周期 |
|---|---|---|
| JWT token | `localStorage`(key `ideabox.jwt`) + module-level cache(避免每次读) | 7 天,过期自动 re-login |
| 项目列表 | `useProjects()` hook 内 `useState` + 路由切换时重拉 | 当前会话 |

**无乐观更新**(MVP 简化):写入 → 等响应 → 更新 state。响应慢时按钮短暂 disabled。

### 4.4 API 客户端契约

`web/lib/api.ts` 暴露:
```typescript
export async function api<T>(
  path: string,
  init?: RequestInit
): Promise<T>;  // 自动加 Authorization + 解析 Result + 401 自动 re-login 一次
```

错误处理:
- HTTP 2xx + `result.code === 0` → 返回 `result.data`
- HTTP 2xx + `result.code !== 0` → throw `ApiError(code, message)`
- HTTP 4xx/5xx → throw `ApiError(httpStatus, raw text)`
- 401001/401002/401003 → 一次性自动 re-login,再失败抛 `ApiError`

### 4.7 部署 / 回滚边界

#### Dockerfile(三阶段,Outside voice A/B 修法)

```dockerfile
# Stage 1: deps
FROM node:22-alpine AS deps
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci --no-audit --no-fund

# Stage 2: builder(NEXT_PUBLIC_* 必须 build-time inline)
FROM node:22-alpine AS builder
WORKDIR /app
ARG NEXT_PUBLIC_API_BASE_URL
ARG NEXT_PUBLIC_AUTH_MODE
ENV NEXT_PUBLIC_API_BASE_URL=$NEXT_PUBLIC_API_BASE_URL
ENV NEXT_PUBLIC_AUTH_MODE=$NEXT_PUBLIC_AUTH_MODE
COPY --from=deps /app/node_modules ./node_modules
COPY . .
ENV NODE_ENV=production
RUN npm run build

# Stage 3: runner — standalone 模式 3 处 COPY 不能漏
FROM node:22-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production PORT=3000 HOSTNAME=0.0.0.0
RUN addgroup --system --gid 1001 nodejs && adduser --system --uid 1001 nextjs
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static
COPY --from=builder --chown=nextjs:nodejs /app/public ./public
USER nextjs
EXPOSE 3000
CMD ["node", "server.js"]
```

**build 命令**:`docker build --build-arg NEXT_PUBLIC_API_BASE_URL=https://api.idea.example.com --build-arg NEXT_PUBLIC_AUTH_MODE=wechat -t ideabox-web:tag ./web`

**关键约束**(Outside voice A):
- `NEXT_PUBLIC_*` **必须 build-time inline**,`docker run -e` 改不动 bundle
- 每环境(dev / staging / prod)各 build 一个 image,**不可共享镜像**
- V5 验证 prod 镜像必须用 `--build-arg` 传 prod URL,curl `/_next/static/<chunk>.css` 确认 200(防 standalone 漏拷)

#### nginx 反代(Outside voice C,新增 I4a 任务)

```nginx
# nginx.conf 片段
server {
  listen 80;
  server_name idea.example.com;

  # CSP for XSS mitigation(ADR-18)
  add_header Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; connect-src 'self'; frame-ancestors 'none';" always;
  add_header X-Frame-Options DENY always;
  add_header X-Content-Type-Options nosniff always;

  # API 反代到 backend
  location /api/ {
    proxy_pass http://backend:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }

  # 一切其他到 Next.js
  location / {
    proxy_pass http://web:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
}
```

### 4.8 风险

| 风险 | 概率 | 应对 |
|---|---|---|
| Next.js 16 + Turbopack Windows dev 不稳 | 中 | 降级到 webpack(`next dev` 去 turbopack) |
| Tailwind 4 alpha → 还在变 | 低 | 锁版本 `4.0.0` 或降 3.x |
| `localStorage` 在 SSR 拿不到 → hydration mismatch | 中 | 严格 `'use client'` + useEffect 内取 |
| CORS preflight 失败 | 低 | PR-1 已配 `CorsConfig`;dev 用 `http://localhost:3000` origin |
| dev login 暴露到 prod | 中 | `@Profile("dev")` 已限,prod profile 启动时 endpoint 不存在;**前端要看环境变量决定是否走 dev login**(prod 走 PR-4 micro auth) |
| @dnd-kit 拖拽期间网络请求失败 → 顺序回滚 UX 不连续 | 中 | 乐观更新本地 sortOrder + 异步 `PUT /reorder`;失败 toast 报错 + revert 到原顺序;失败重试按钮 |
| 拖拽 vs 点击 ✓ vs 双击编辑 手势冲突 | 中 | @dnd-kit `activationConstraint: { distance: 8 }`(拖动 8px 才算拖拽,< 8 算点击);拖拽手柄独立 grip icon(不在卡片正文区) |
| 触屏端长按编辑 vs 拖拽冲突 | 中 | mobile 拖拽要求长按 200ms + 拖 8px;500ms 触发编辑(若 200-500 之间松手则不动作);用 @dnd-kit `PointerSensor` + `KeyboardSensor` |
| Dockerfile standalone 漏 COPY `.next/static` / `public/` | 中 | 三处 COPY 显式写出(§ 4.7);V5 必须 curl 一个 `_next/static/<chunk>.css` 验证 200 |
| `NEXT_PUBLIC_*` build-time inline 误用 runtime env | 中 | 每环境 build 各自镜像;V5 用 `--build-arg` 显式传;镜像不可跨环境共享 |
| localStorage JWT XSS 风险 | 中 | ADR-18 四件套:CSP / ESLint no-danger / textContent / idle 30min auto-clear;PR-4 切 cookie 模式 |
| Next.js 16 + Tailwind 4 + @dnd-kit + SWR + next/font 五件叠加 Windows 首次跑通 | 中 | dev 优先单组件冒烟(I1-I3 跑通基础),逐项加(F1 加 fetch / F4 加 @dnd-kit / I5 加 SWR),避免一次性引入难定位 |
| Noto Sans SC self-host 字体文件大小 ≈ 1.5MB | 低 | `next/font/google` 自动 subset CJK 常用 5500 字 + WOFF2 压缩到 ~400KB;启用 `font-display: swap` |
