## 3. 已知风险 / 不测项

### 3.1 已知风险(接受)

- **postcss CVE GHSA-qx2v-qp2m-jg93**(CVSS 6.1 moderate):build-time only,prod runtime 不暴露;`globals.css` 是 trusted source。`/cso` daily 8/10 gate 已 suppress。后续 PR 跟踪 postcss 升级。
- **localStorage JWT XSS 暴露面**:ADR-18 四件套缓解(CSP `script-src 'self' 'unsafe-inline'` + ESLint `react/no-danger:error` + textContent 渲染 + idle 30min auto clearToken)。**PR-4 切 httpOnly cookie 模式**。
- **nginx CSP `'unsafe-inline' 'unsafe-eval'`**:Next.js 16 RSC + hydration 强制要求(无 nonce middleware 无法 hydrate)。长期 TODO 接 nonce middleware,详 `nginx.conf` 注释。
- **删除时卡片无视觉反馈**:实施 deviation,ADR-10 设计是"卡片淡出 + 高度收缩 0.3s",当前仅 toast 提示 + 5s 后真删卡片消失。PR-3.x hotfix 处理。

### 3.2 不测项(透明声明)

- **T5 Playwright e2e 5 case**:推迟到 PR-3.x hotfix 或 PR-4 一并做。阶段 5 Claude in Chrome 真浏览器联调 13 金路径 + 7 边界已实质覆盖等价 e2e。
- **Lighthouse a11y ≥ 90**:本机 prod 预演时验证,本文档未跑数(SKILL 联调阶段限制 viewport resize)。PR-4 上线前必跑。
- **iOS Safari / Android Chrome 真机测试**:开发期无真机,Chrome DevTools device mode 模拟。PR-4 上线后用户反馈验证。
- **移动端 < 768px 单栏布局 + AppBar + 抽屉**:UI 设计已规划,实施不在 PR-3 scope(响应式 CSS 通用,但 AppBar / 抽屉组件未独立),桌面端联调 ✅,移动端实际渲染留下个 PR 验证。
- **跨用户 IDOR 测试**:后端 PR-1 hotfix 已校验 `getOwnedById` 路径 + 集成测试覆盖;前端无需重复测。
- **大量数据性能**:100+ 想法、20+ 项目场景未模拟。MVP 数据量小不阻塞;`/cso` 评估也未列为 P0/P1。
