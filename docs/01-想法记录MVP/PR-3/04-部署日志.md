## 5. 本次踩坑

### 坑 1:Spring Security `CorsConfigurer` 优先按 bean name `corsConfigurationSource` 查找

- **现象**:前端跑 3001 端口时所有 OPTIONS preflight 403 "Invalid CORS request",哪怕 `allowedOriginPatterns: ["*"]`
- **根因**:`CorsConfigurer.getCorsConfigurationSource(http)` 先按 bean name `corsConfigurationSource` 查 ApplicationContext,再 fallback `getBeansOfType`。我们 bean name `devCorsConfigurationSource` fallback 路径未生效
- **修法**:`@Bean(name = "corsConfigurationSource")` 显式指定 bean name
- **教训**:**真浏览器 e2e 联调暴露 PR-1 漏 bug**;单测 MockMvc 不走真 CORS filter chain。回流 PR-1 沉淀到 _开发踩坑记录.md

### 坑 2:本机 PG 25432 / Redis 26380 端口规划

- **现象**:用户机器其他项目(sosogithub)占了 15432 / 6380
- **修法**:改 25432 / 26380(已同步改 docker-compose.dev.yml / .env / application.yml / IntegrationTestBase / 设计文档 5 处)
- **教训**:多项目机器**端口预约表**有用 — 项目级 README 应记录"本项目占用端口"

### 坑 3:framer-motion `layout` prop 与 dnd-kit transform 冲突 → 拖拽动画"跳"

- **现象**:`motion.div` + `layout` 在 IdeaItem 上,拖拽时卡片"先回到原位,等 300ms 才跳到新位置"
- **根因**:framer-motion `layout` 用自己的 transform/transition 覆盖了 dnd-kit 的 `style.transform` 和 `style.transition`
- **修法**:移除 `motion.div` 改回 `<div>`,用 dnd-kit 自带 `defaultAnimateLayoutChanges`,framer-motion 仅在 Toast/Modal/EmptyState/SVG 等独立组件用
- **教训**:dnd-kit 内部用 transform 做 layout,**不能再叠加 framer-motion layout**;两者职责互斥

### 坑 4:拖完后本地顺序 debounce 300ms 才更新 → 看起来"回弹再跳"

- **现象**:用户拖完想法,卡片闪回原位,300ms 后跳到新位置
- **根因**:`scheduleReorder` 把整个 `onReorder` 操作 debounce,包括本地 SWR cache 的乐观更新
- **修法**:拆 `onLocalReorder`(立即调,更新本地)+ `onApiReorder`(debounced 300ms 调 API)
- **教训**:**乐观更新本地 cache 永不 debounce**;debounce 仅针对网络请求

### 坑 5:framer-motion AnimatePresence exit 在 jsdom 不触发

- **现象**:ToastProvider 测试 "info 3s 后自动消失"在 jsdom 中失败(toast 仍在 DOM)
- **根因**:jsdom 无完整 requestAnimationFrame 循环,framer-motion AnimatePresence exit 动画异步不完成
- **修法**:测试用例 `it.skip` 透明声明 + 浏览器 e2e 验证
- **教训**:**jsdom 不可测 framer-motion 异步动画的卸载**;改用 e2e

### 坑 6:Maven `mvn spring-boot:run` 跳过编译 → 改源码后旧 .class 仍生效

- **现象**:改 CorsConfig.java 后重启 backend,行为不变 — `Nothing to compile - all classes are up to date`
- **修法**:`rm target/classes/com/ideabox/api/config/CorsConfig.class` 强制重编,或先 `mvn compile`
- **教训**:dev 改 Java 源码后 spring-boot:run 不保证看到新代码;必要时手动 clean target
