# 0X-功能名 · 开发计划

> 对应 SOP 阶段 5。把功能拆成最小粒度 PR，每个 PR 独立可上线、可回滚。
> 必须经用户认可拆分后才能进入实现阶段。

---

## 1. PR 拆分

| PR | 范围 | 风险 | 估时 | 上线策略 |
|---|---|---|---|---|
| PR1 | （eg. 新建表 + 后端只读接口） | 低 | 半天 | 直接上 |
| PR2 | （eg. 前端只读页面接入） | 低 | 半天 | 直接上 |
| PR3 | （eg. 写入接口 + 副作用） | 中 | 1 天 | 灰度 / feature flag |
| ... |  |  |  |  |

**总体上线顺序**：PR1 → PR2 → PR3，每个 PR 上线后观察 X 小时再上下一个。

---

## 2. 每个 PR 的回滚边界

- **PR1 回滚**：drop 新表（数据无价值时）/ 保留表关掉接口
- **PR2 回滚**：前端单 commit revert
- **PR3 回滚**：feature flag 关掉
