# 功能模板用法说明

> 新功能起步：`cp -r docs/_功能模板/ docs/<NN>-<功能名>/`，然后按本说明填写。

## 目录结构

```
docs/<NN>-<功能名>/
├── 01-设计.md              # feature 级：user story / 路线图 / 跨 PR ADR / 度量
├── 02-开发计划.md          # feature 级：PR 总览表 + 跳链 → 各 PR 子目录
├── 05-迭代日志.md          # feature 级：PR 完工时间线索引
├── 资源/                    # 共享资源（线框图 / 设计规范）— 可选
├── PR-N/                   # ← 每个 PR 一个子目录（启动该 PR 时建）
│   ├── 01-设计.md          # PR-N user story + UI 规格 + 调研
│   ├── 02-开发计划.md      # PR-N 任务分解（B/F/I）
│   ├── 03-测试.md          # PR-N 测试计划 + 报告
│   ├── 04-部署日志.md      # PR-N 部署快照（8 节 SOP：改动汇总 / 部署步骤 / 回滚 / 踩坑 / commit / 测试覆盖 / 观察清单）
│   └── 05-迭代日志.md      # PR-N 完工 + hotfix 时间线
└── ...
```

## 工作流（与 piao-workflow skill 对齐）

1. **新功能启动**：`cp -r docs/_功能模板/ docs/<NN>-<功能名>/`
2. **阶段 0 需求确认**：填外层 `01-设计.md § 1` + `02-开发计划.md § 1 PR 总览表`（标各 PR 范围 / 风险 / 估时）
3. **某 PR 启动时**：`cp -r docs/<NN>-<功能名>/PR-N/ docs/<NN>-<功能名>/PR-1/` （把 PR-N 模板复制为对应实际 PR 号）
4. **PR 内 SOP**：填该 PR 子目录的 `01-设计.md` / `02-开发计划.md` / `03-测试.md` / `04-部署日志.md` / `05-迭代日志.md`
5. **PR 完工后**：在外层 `02-开发计划.md § 1 PR 总览表` 该行加 ✅ ship 标记 + commit 链接 / 在外层 `05-迭代日志.md` 顶部加一行时间线索引

## 编号约定

- **外层文档**（feature 级）：标题用纯 `## 1. / 2. / ...` SOP 编号
- **PR 子目录内文档**（PR 级）：标题用纯 `## 1. / 2. / ...` SOP 编号（因子目录路径已隔离命名空间，无需 `<N>-PR<X>.` 前缀）

## 远期 PR

外层 `02-开发计划.md § 1 PR 总览` 列出所有规划中的 PR 概要。**远期未启动的 PR 不建子目录**（等启动时再建，避免空目录污染）。

## 跨 PR 共享内容归属

- **跨 PR user story / 度量** → 外层 `01-设计.md`
- **跨 PR ADR**（多 PR 都用的决策）→ 外层 `01-设计.md § 4`
- **跨 PR 调研**（多 PR 都依赖的技术调研）→ 外层 `01-设计.md § 2`
- **跨 PR UI 共享规范**（如设计 token）→ 外层 `01-设计.md § 3` 或独立的 `资源/UI设计规范.md`
- **PR 特定 user story / 调研 / UI 规格** → 对应 `PR-N/01-设计.md`
- **PR 特定任务分解** → `PR-N/02-开发计划.md`
- **PR 特定踩坑** → `PR-N/04-部署日志.md § 5`（跨功能可复用的坑提升到根目录 `docs/_部署踩坑记录.md` / `_开发踩坑记录.md`）
